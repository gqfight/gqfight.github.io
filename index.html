NPM模块管理

安装完成node后，基本自带npm模块管理器

我们需要一个第三方（别人写的）模块，插件，类库或者框架，需要提前下载才可以使用

基于npm第三方包管理器下载（yarn/bower。。。）

1.npm下载的资源在http://www.npmjs.com中下载的

npm install xxx

npm install xxx -g

npm uninstall xxx npm uninstall xxx -g

把资源或者第三方模块安装到全局环境下（目的：以后可以基于命令来操作一些事情）

基于npm安装的一些细节点

需要联网

下载成功后当前目录中多增加一个node_modules

一般来说，下载下来的内容包含源码和最后供开发者使用的版本

解决下载慢的问题

基于nrm切换到国内下载源

首先安装nrm，而且是把它安装到全局环境下

安装完成后可以使用两个命令

nrm ls 查看当前可用源

nrm use xxx 使用某个源

切完源还是基于npm安装操作

可以基于yarn来安装管理

安装先安装yarn到全局，然后基于yarn安装我们需要的模块

yarn add xxx 安装

yarn remove 移除

只能安装在本地

基于cnpm淘宝镜像来处理

解决安装版本问题

首先查看当前模块的历史版本信息

npm view jquery > jquery.version.json

使用npm view jquery versions

这种方式可以查看npm服务器上所有的jquery版本信息；

把当前模块的历史信息输出到具体的某个文件中

安装指定版本

yarn add jquery@1.11.3:npm和yarn 都是这样

安装bower 从github 全局安装less /babel-cli
